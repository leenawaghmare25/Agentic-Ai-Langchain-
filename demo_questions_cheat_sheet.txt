--------------------------------------------------------------------------------

QUESTION 2: "How do these agents work in a loop?" (The ReAct Loop)
--------------------------------------------------------------------------------
★ Easy Explanation:
  The agents run in a cycle loop (up to 15 times). On each turn, the agent:
  1. Thinks about what needs to be done.
  2. If it decides it needs to edit a file, it calls the "write_file" tool.
  3. Our Python script runs the tool locally, gets the success result (observation),
     and appends it back to the agent's memory (scratchpad).
  4. The agent repeats this until it decides the files are complete, then it finishes.

★ Code to Show:
  - The Loop Statement: Line 998 (for cycle in range(1, 15):)
  - Tool Invocation Code: Line 1048 (observation = target_tool.invoke(tool_args))
  - Appending Tool Results to Memory: Lines 1060 - 1061
    Code:
      fe_scratchpad.append(AIMessage(content=content, tool_calls=[tc]))
      fe_scratchpad.append(ToolMessage(content=observation, tool_call_id=tc.get("id")))

QUESTION 3: "How does the output of one agent become the input of the next?"
--------------------------------------------------------------------------------
★ Easy Explanation:
  We use a plain text string accumulator variable called "pipeline_context".
  As each agent finishes, its output report is added directly to "pipeline_context".
  This string is then passed as the "human" input message to the next agent in the sequence.

QUESTION 5: "How does the fallback system work when a model fails?"
--------------------------------------------------------------------------------
★ Easy Explanation:
  When invoking the AI, we use a try-except loop. If the active model raises an error
  (e.g., API limit, credit limit, or timeout), our code catches the exception and
  automatically switches the provider name to the next backup model listed in our
  "fallback_ladder" list, then retries.

★ Code to Show:
  - Fallback Ladder List: Line 737 (fallback_ladder = ["Gemini", "Groq", "Ollama_7b", "Ollama_1.5b"])
  - Auto-Fallback Switching: Lines 773 - 779 (inside invoke_agent_with_fallback)
    Code:
      current_idx = fallback_ladder.index(provider)
      provider = fallback_ladder[current_idx + 1]
      continue

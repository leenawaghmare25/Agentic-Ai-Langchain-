### 1. Architecture Decision

The application will follow a **Client-Server architecture** with a **Single-Page Application (SPA)** frontend.

*   **Frontend (Client-Side):** A single `index.html` page rendered by Flask, which then uses vanilla JavaScript to dynamically interact with the backend API. All UI logic, user input handling, and display updates occur here.
*   **Backend (Server-Side):** A lightweight Flask server acting as an API provider. It will handle password generation requests and return the results as JSON.
*   **Communication:** The frontend will communicate with the backend using standard RESTful HTTP `GET` requests via the `fetch` API.

**Data Flow:**
1.  User loads `index.html` in their browser.
2.  User adjusts slider for length and checks/unchecks options for numbers/symbols.
3.  User clicks "Generate Password" button.
4.  `script.js` gathers the current UI parameters.
5.  `script.js` sends an asynchronous `fetch` request to the Flask backend's `/generate` endpoint with these parameters as query strings.
6.  Flask's `app.py` receives the request, parses parameters, and securely generates a password using Python's `secrets` module.
7.  Flask returns the generated password in a JSON response.
8.  `script.js` receives the JSON response, extracts the password, and updates the UI to display it.
9.  The "Copy to Clipboard" button becomes active, allowing the user to copy the displayed password.

### 3. APIs

**Endpoint:** `/generate`

*   **Method:** `GET`
*   **Description:** Generates a secure password based on user-defined criteria.
*   **Query Parameters:**
    *   `length` (Integer, **required**): The desired length of the password.
        *   **Constraints:** Must be an integer between 8 and 32 (inclusive).
        *   **Example:** `length=16`
    *   `include_numbers` (Boolean, optional): Specifies whether to include numeric characters (0-9).
        *   **Default:** `false`
        *   **Example:** `include_numbers=true`
    *   `include_special_characters` (Boolean, optional): Specifies whether to include common special characters.
        *   **Default:** `false`
        *   **Example:** `include_special_characters=true`

*   **Response (Success - HTTP 200 OK):**
    ```json
    {
        "password": "YourGeneratedPassword123!"
    }
    ```

*   **Response (Error - HTTP 400 Bad Request):**
    *   Returned if `length` is missing, not an integer, or outside the 8-32 range.
    *   Returned if `include_numbers` or `include_special_characters` are provided but cannot be parsed as a boolean (e.g., not "true" or "false").
    ```json
    {
        "error": "Invalid parameter: length must be an integer between 8 and 32."
    }
    ```

*   **Response (Error - HTTP 500 Internal Server Error):**
    *   Returned for any unexpected server-side issues during password generation.
    ```json
    {
        "error": "An unexpected error occurred on the server."
    }
    ```

### 4. Backend Server Framework

*   **Framework:** Flask
*   **Core Logic:** The `app.py` file will instantiate the Flask application, define the `/generate` route, implement the password generation logic using `secrets.choice()` and `string` constants, and handle parameter validation and JSON responses.
*   **Database Rule Adherence:** The application does not require a database. Therefore, the `with app.app_context(): db.create_all()` pattern is not directly applicable. However, if a database were introduced in the future, this critical rule would be strictly followed for any database initialization or schema creation, ensuring modern Flask practices.

---

### 5. Module Structure (Backend - Python)

For this application's scope, a single `app.py` file is sufficient and adheres to Flask's common practices for smaller projects.

*   **`app.py`**:
    *   **Imports:** `Flask`, `request`, `jsonify`, `secrets`, `string`.
    *   **Flask App Initialization:** `app = Flask(__name__)`.
    *   **Route Definition:** `@app.route('/generate', methods=['GET'])` for the password generation endpoint.
    *   **Parameter Parsing & Validation:** Logic to extract `length`, `include_numbers`, `include_special_characters` from `request.args` and validate their types and ranges.
    *   **Password Generation Logic:** A function (or inline logic) that constructs the character pool based on user options and uses `secrets.choice()` to build the password.
    *   **JSON Response:** `jsonify()` to return the generated password or error messages.
    *   **Standard Execution Block:** The `if __name__ == "__main__": app.run(debug=True)` block at the very end.
